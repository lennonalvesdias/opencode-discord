# 🤝 Contribuir para o RemoteFlow

Primeiro, obrigado por considerares contribuir para o **RemoteFlow**! É o apoio da comunidade que transforma pequenos projetos em ferramentas incríveis.

Para garantir uma colaboração fluida, por favor, segue estas diretrizes.

## 📝 Código de Conduta

Ao participar neste projeto, concordas em manter um ambiente respeitoso, inclusivo e livre de assédio para todos.

## 🛠️ Como Posso Contribuir?

### 1. Reportar Bugs

Encontrou algo que não funciona?

- Verifica se já existe uma *Issue* aberta sobre o problema.
- Se não, abre uma nova *Issue* detalhando o erro, passos para reproduzir e o seu ambiente (SO/Node version).

### 2. Sugerir Funcionalidades

Tem uma ideia para tornar o fluxo remoto ainda melhor?

- Abre uma *Issue* com a tag `enhancement`.
- Explica o "porquê" da funcionalidade e como ela ajudaria outros utilizadores.

### 3. Enviar Pull Requests (PRs)

1. Faz um **Fork** do repositório.
2. Cria uma **Branch** para a tua alteração (`git checkout -b feature/nome-da-feature`).
3. Mantém o estilo de código existente.
4. Garante que a documentação é atualizada, se necessário.
5. Envia o **Pull Request** para a branch `main`.

## 💻 Configuração de Desenvolvimento Local

Para testar as tuas alterações:

1. Segue os passos de instalação no [README.md](./README.md).
2. Recomendamos o uso de um servidor (Guild) de testes no Discord.

## 🎨 Padrões de Commit

Seguimos a convenção de [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` para novas funcionalidades.
- `fix:` para correção de bugs.
- `docs:` para alterações na documentação.
- `refactor:` para melhorias no código sem alterar funcionalidade.
